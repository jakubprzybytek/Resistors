# Implementation Plan — Better Ranking (balanced anchor + two-way spine)

## Summary
Replace the flat, paginated result list and the `accuracy` / `resistorCount` toggle with a **spine-based navigation model**. On `Calculate` we compute the same ±5% / 5-resistor result set as today, deduplicate it by a canonical network form, group results into **complexity tiers** (one per resistor count), and reduce those tiers to the **Pareto frontier** (the "spine"). The user starts at a **balanced anchor** (the knee of the spine) and moves **more accurate** (up, more resistors) or **simpler** (down, fewer resistors), revealing adjacent tier cards. Each tier card is a **carousel** over that tier's alternative configurations, ordered by ascending `absError` and bounded so it never becomes worse than the next simpler tier's best.

Decisions locked in from story review: knee heuristic for the anchor (#1), similar ordering is `absError` only (#2), no reset control (#3). Search completeness stays deferred — the model is built on whatever set the search returns.

## Target architecture
- **`src/network.ts`** (updated): add a pure `canonicalKey(node)` helper that produces a structural normal form (flatten associative series/parallel chains, sort operands, round leaf values) so duplicates like `A + (B ∥ C)` and `(C ∥ B) + A` collapse. `findAllResistorNetworks` is otherwise unchanged; it still returns the pruned ±5% set.
- **`src/resultRanking.ts`** (rewritten, pure module): drop `RankingMode` / `sortResults`. Export the new model types and builders:
  - `TierCard` (a spine tier: `count`, `bestError`, and its bounded, error-sorted `results: NetworkResult[]`).
  - `RankingModel` (`tiers: TierCard[]` ordered most-accurate → simplest, plus `anchorIndex`).
  - `buildRankingModel(results, opts)` — dedup, tier, Pareto-filter, bound the similar sets, and select the knee.
  - `selectKnee(tiers)` — knee/elbow detection returning the anchor index.
- **`src/App.tsx`** (updated): on `Calculate`, run the search, log count/time (unchanged), call `buildRankingModel`, store the model plus a `calcId` that increments each calculation (used to remount the view so navigation state resets on recalculation — the only reset, per decision #3).
- **`src/components/RankingView.tsx`** (new, replaces `ResultsList`): owns the reveal window (`topIndex`/`bottomIndex`, both initialised to `anchorIndex`) and per-tier carousel positions. Renders a "Show more accurate" control above the column and a "Show simpler" control below it, with the revealed `TierCard`s stacked between them. Handles the empty state.
- **`src/components/TierCard.tsx`** (new): renders one spine tier — header (resistor count + current deviation), the current configuration via `ResultCard`, left/right carousel arrows, and a `n / m` position indicator.
- **`src/components/ResultCard.tsx`** (unchanged or minimally touched): still renders a single `NetworkResult`.

## Phase 1 — Canonical dedup (`network.ts`)
1. Add `export function canonicalKey(node: Node): string`:
   - Leaf → `L<roundedValue>` (round to a shared precision constant to absorb FP noise).
   - Series/parallel → flatten same-kind children into a flat operand list (`(A + B) + C`, `A + (B + C)`, `A + B + C` all become one series group of `{A, B, C}`), recurse `canonicalKey` on each operand, sort the keys, and wrap: `S(<sorted keys>)` / `P(<sorted keys>)`.
2. Export a single precision constant reused by dedup, tier-best selection, and the similar bound (Phase 2).
3. Do **not** change `findAllResistorNetworks` search behaviour; it still returns the pruned ±5% set (completeness deferred).

## Phase 2 — Ranking model (`resultRanking.ts`, rewrite)
1. Remove `RankingMode` and `sortResults`.
2. Types:
   - `interface TierCard { count: number; bestError: number; results: NetworkResult[] }` — `results` sorted ascending by `absError`, already bounded.
   - `interface RankingModel { tiers: TierCard[]; anchorIndex: number }` — `tiers[0]` is the most accurate (highest count), last is the simplest (lowest count).
3. `buildRankingModel(results: NetworkResult[], opts: { relTolerance?: number; target: number }): RankingModel`:
   1. **Dedup** by `canonicalKey(result.node)`, keeping one representative per key.
   2. **Group** the deduped results by `count`.
   3. **Tier best** = min `absError` per count.
   4. **Pareto spine**: iterate counts ascending; keep a count only if its tier-best is strictly lower than the best tier-best kept so far (drop dominated tiers entirely).
   5. **Order** the kept tiers most-accurate → simplest (descending count).
   6. **Bound the similar set** for each kept tier at count `C`: keep configs with `absError` in `[tierBest(C), min(E_next, tolEdge))`, where `E_next` is the next simpler kept tier's `bestError` and `tolEdge = relTolerance * target`; if there is no simpler tier, use `tolEdge`. Sort each tier's kept configs ascending by `absError` (decision #2: `absError` only, no secondary key).
   7. `anchorIndex = selectKnee(tiers)`.
4. `selectKnee(tiers: TierCard[]): number`:
   - `tiers.length <= 1` → `0`.
   - `tiers.length === 2` → the more accurate endpoint (index `0`).
   - Otherwise: normalise each tier's `(count, bestError)` to the unit square, take the straight line between the two endpoints, and return the index with the greatest perpendicular distance from that line (Kneedle-style elbow). Break ties toward the more accurate tier (lower index).

## Phase 3 — `App.tsx` wiring
1. State: `model: RankingModel | null`, `target: number`, `hasCalculated: boolean`, `calcId: number`.
2. `handleCalculate(values, target)`:
   - `performance.now()` around `findAllResistorNetworks(values, target, { relTolerance: 0.05, maxResistors: 5 })`.
   - Keep the two existing `console.log` lines (found count, computation time).
   - `setModel(buildRankingModel(found, { relTolerance: 0.05, target }))`, `setTarget(target)`, `setHasCalculated(true)`, `setCalcId(id => id + 1)`.
3. Render `{hasCalculated && <RankingView key={calcId} model={model} target={target} />}` directly below `ResistorForm` (remounting via `key` resets navigation on each new calculation — the only reset, per decision #3).

## Phase 4 — `RankingView` component
1. Props: `model: RankingModel | null`, `target: number`.
2. Empty state: if `model` is null or `model.tiers.length === 0`, render the existing empty-state block (`role="status"`) with the "no results could be determined…" copy.
3. Navigation state (local `useState`, reset by remount): `topIndex` and `bottomIndex`, both initialised to `model.anchorIndex`; `positions: number[]` of length `tiers.length`, initialised to `0`.
4. Layout, top to bottom:
   - **"Show more accurate"** button — visible/enabled only when `topIndex > 0`; click → `topIndex--`.
   - The revealed tiers `tiers[topIndex..bottomIndex]` (most accurate first), each rendered as `<TierCard>` with its carousel `position` and prev/next handlers that update `positions[i]`.
   - **"Show simpler"** button — visible/enabled only when `bottomIndex < tiers.length - 1`; click → `bottomIndex++`.
5. Only the anchor card shows initially (both indices equal `anchorIndex`); revealed cards persist as the user navigates.

## Phase 5 — `TierCard` component
1. Props: `tier: TierCard`, `position: number`, `onPrev`, `onNext`, `target`.
2. Renders: a header with resistor count and the current config's deviation; `<ResultCard result={tier.results[position]} target={target} />`; **left arrow** (disabled at `position === 0`) → `onPrev`; **right arrow** (disabled at `position === tier.results.length - 1`) → `onNext`; a `position + 1 / tier.results.length` indicator.
3. Left = next non-worse (lower `absError`), right = next not-better (higher `absError`), per rule 5a.

## Phase 6 — Styling

2. `TierCard.scss`: `.tier-card` container, `.tier-card__carousel` row (arrows + card), `.tier-card__position` indicator, disabled-arrow styling.

## Migration notes
- `RankingMode` / `sortResults` and their tests are removed; `ResultsList.tsx`/`.scss` and `ResultsList.test.tsx` are replaced by `RankingView` + `TierCard`; `more-results.spec.ts` is removed. Update any imports in `App.tsx` accordingly.

## Resolved decisions
1. Anchor = **knee** of the spine (Kneedle-style elbow); length-1/2 spines use the more accurate endpoint. (Story open question 1.)
2. Similar ordering within a tier = ascending `absError` only, no secondary tie-break. (Story open question 3.)
3. No reset/collapse control; navigation state resets only when a new `Calculate` remounts the view via `calcId`. (Story open question 6.)
4. Dominated tiers are dropped entirely (rule 2).
5. Similar set bounded to `[tierBest, min(nextSimplerBest, ±5% edge))`; smallest tier falls back to the ±5% edge (rule 3).
6. Search completeness stays deferred — the model is built on the current pruned result set.
