Show more than one result for user to pick from

* Each calculation should compute all unique results whose deviation is within +/-5% of the target value.
* The initial result list should show the first three results.
* There should be a button `Show 3 more` that appends the next three unseen results.
* If fewer than three unseen results remain, `Show 3 more` should append all remaining results.
* Results should be shown one under another.
* The results list should be displayed below the `Calculate` button.
* The results area should contain a toggle that switches ranking between:
* accuracy ascending (lowest absolute deviation first; ties are broken by lower resistor count)
* number of resistors ascending (lowest resistor count first; ties are broken by lower absolute deviation)
* `Calculate` should compute the full result set once, so switching the ranking only reorders the already computed results.
* If two results are tied on both ranking properties, they may be shown in any order.
* Switching the ranking type should reset pagination back to the first three results.
* Clicking `Calculate` should reset previously shown results and pagination, then show the first three results for the current inputs.
* Changing any input values before clicking `Calculate` should not update the currently displayed results until `Calculate` is clicked again.
* The calculation should search combinations using a maximum of 5 resistors.
* Results should be unique by resistor combination. Two results with the same achieved value but different resistor combinations should both be shown.
* No valid result within the search constraints should be omitted from the computed result set.
* Each calculation should log the total number of found results to the developer console.
* Each calculation should log the computation time to the developer console.
* If no results can be determined within the search constraints, the results area should show a message explaining that no results could be determined for the given inputs.
* The default ranking should be accuracy.

Minimum acceptance criteria

* On `Calculate`, show the first three unique results from the full +/-5% result set.
* On `Show 3 more`, append the next three unseen results using the current ranking.
* Changing ranking should reorder the same computed result set and reset the visible list to the first three results.
* Changing any inputs and clicking `Calculate` should recompute the full result set and reset pagination.
* The results list should be rendered below the `Calculate` button.
* When no more unseen results remain, hide or disable `Show 3 more`.
* On each `Calculate`, log the number of found results and the computation time in the developer console.
* Accuracy ranking should use resistor count as the tie-breaker; resistor-count ranking should use absolute deviation as the tie-breaker.
* Use a maximum of 5 resistors when computing the result set.
* If no results are found within the search constraints, show an explanatory empty-state message.
