In addition to providing available resistor values, there should be an option to pick standard series like E3, E6, E12, E24.

* around available resistor values, there should be tabs: E3, E6, E12, E24, Custom
* predefined resistor value series (E3-E24) should be hardcoded (not editable) and list appropriate values (for E3 there will be: 1, 2.2, 4.7, 10, 22, 47, 100 etc., up to 100 MOhm). It means that selecting a given tab should put appropriate values into the text area.
* predefined values in textarea should use suffix tokens and be grouped by decade rows for all predefined tabs (one decade row per line, e.g. base values, then x10, then x100, etc.). Example for E3:
** line 1 (no suffix): 1, 2.2, 4.7
** line 2 (no suffix): 10, 22, 47
** line 3 (`k` suffix): 1k, 2.2k, 4.7k
* textarea for tabs with predefined value series should be in read-only mode
* custom tab should allow providing resistor values by hand:
** comma or newline separated values
** `k` and `M` suffixes supported (`k` means kilo, 1000; `M` means mega, 1000000). Suffixes are case-insensitive. There can be a space between value and suffix or not. '4.7k' means 4700, '1 k' means 1000.
** custom values should be stored in memory so that when moving away from it and getting back, the user should see previous values. Refreshing the page should wipe them out.
** invalid values should produce a validation error explaining which value is incorrect
** presence of any invalid token should block calculation
* 'Calculate' button works on values from active tab only
* first tab is active by default

Decade values
* min value: 1 Ohm
* max value: 100 MOhm
